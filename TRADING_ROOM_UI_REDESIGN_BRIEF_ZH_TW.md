# IUF 交易戰情室 UI 改版簡報

## 目標

把目前偏「乾淨 SaaS 控制台」的視覺，升級成更像台股操盤桌會真的想長時間盯盤的「交易戰情室」。

改版重點不是單純深色化，而是：

- 全面改成繁體中文介面
- 提高資訊密度，但維持層級清楚
- 強化盤中節奏感、風險感、狀態感
- 讓 `主題 -> 公司 -> 訊號 -> 計畫 -> 檢討 -> 草稿/代理` 有明確作戰流

## 目前問題

1. 配色太像米白設計稿，不像交易室
2. 側欄過寬、主畫面節奏太鬆，資訊密度不足
3. 英文 UI 文案過多，與台股使用情境不一致
4. 卡片樣式偏統一，缺乏「狀態優先」的視覺語言
5. 缺少數字/價格/時間的終端感與指揮感

## 建議視覺方向

主題名稱：
`台股 AI 交易戰情室`

風格關鍵字：

- terminal dashboard
- institutional blotter
- tactical operations room
- dark glass + steel grid
- data-first, not marketing-first

## 色彩系統

### 基底

- 背景：`#07111A`
- 第二背景：`#0C1722`
- 面板：`#111F2D`
- 面板高亮：`#16283A`
- 邊框：`rgba(148, 163, 184, 0.18)`

### 功能色

- 多方/正向：`#13C296`
- 空方/風險：`#FF5D5D`
- 觀察/中性：`#F5B942`
- 主要強調：`#4DA3FF`
- 次要高亮：`#7CFFB2`

### 文字

- 主文字：`#E7EEF7`
- 次文字：`#9FB2C8`
- 弱文字：`#6F8196`

## 字體建議

### 介面字體

- `Noto Sans TC`
- `IBM Plex Sans TC`

### 數字 / 代碼 / 價格

- `IBM Plex Mono`
- `JetBrains Mono`

規則：

- 中文介面用繁體中文
- 價格、ticker、時間、風險報酬比一律使用等寬數字
- 頁首標題不要過大，避免像行銷頁

## 版面方向

### 外框

- 左側導覽縮成窄側欄，保留 logo、workspace、主要模組
- 頁面頂部新增一條「戰情列」
- 戰情列放：工作區、資料模式、盤勢標籤、代理狀態、時間

### 首頁/總覽

- 第一屏不是大標題，而是「今日作戰總覽」
- 上排：盤勢、主題數、待審草稿、活躍訊號、風險事件
- 中排：主題熱區、OpenAlice 狀態、稽核摘要
- 下排：最新訊號流 / 任務流 / 稽核流

### 列表頁

- 以高密度表格為主，不要大面積留白卡片
- 表格列需有狀態色條、badge、hover detail
- 重要欄位固定：狀態、ticker、名稱、分類、優先度、最近更新

## 文案語言

全部改成繁體中文，建議用語：

- Themes -> 主題戰區
- Companies -> 公司資料庫
- Signals -> 訊號雷達
- Plans -> 交易計畫
- Reviews -> 交易檢討
- Drafts -> 草稿審核
- Ops -> 系統戰情
- Control Tower -> 戰情中樞
- Primary Desk -> 主交易桌

## 元件規範

### badge

- 多方：綠底綠框
- 空方/失敗：紅底紅框
- 草稿待審：黃底黃框
- 系統正常：藍底藍框

### 面板

- 降低圓角，從 28px 改到 16-20px
- 陰影更收斂，改成內斂的深色堆疊
- 背景加入細緻網格/掃描線，不要大片奶油白

### 表格

- 採交錯列底色
- hover 顯示次要資訊
- ticker / 數值靠右或等寬對齊

## 動態與節奏

- 更新數字用短促淡入，不做花俏彈跳
- 切頁/展開 drawer 用 160-220ms
- 新訊號/新事件可有很輕的 pulse 提示

## 先做的改版順序

1. 全站語言切成繁體中文
2. 重做全域 tokens：背景、文字、badge、panel、表格
3. 縮窄左側欄，新增頂部戰情列
4. `Companies / Signals / Drafts / Ops` 改成高密度戰情表格
5. 首頁改成「今日作戰總覽」而不是展示型 hero

## 對前端實作的要求

- 保留目前功能與資料流，不重寫 API
- 先做 layout + design token + copy rewrite
- 優先桌面版，再補手機 read-only
- 不要做 AI 風行銷頁視覺
- 不要回到淺色米白控制台風格
