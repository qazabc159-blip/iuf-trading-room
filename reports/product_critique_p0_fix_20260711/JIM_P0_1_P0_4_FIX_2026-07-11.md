## P0-1 вҖ” жҺЁи–ҰвҶ’дәӨжҳ“е®Өж–·йҸҲ

### Root cause (differs from the critique's hypothesis)
The critique's screenshot (`chain_rec_to_room.png`) suspected a session/cookie
handshake failure specific to the handoff path. Real-browser repro against
**prod** with a real Owner session (both a direct `/portfolio` open and a
handoff URL with full params: `ticker/prefill/from_rec/entry/stop/tp/side`)
showed the *same* behavior in *both* cases вҖ” the trading room always starts
from an SSR "fast shell" placeholder (`baseCapitalTWD: null` by design, so
first paint never blocks on slow backend calls вҖ” see
`app/api/ui-final-v031/[screen]/route.ts`'s `fastPaperShell`), then a client
script (`clientPaperPayload()`) fetches everything and replaces it.

Network waterfall against prod showed `GET /api/v1/paper/portfolio` itself
resolves in ~1.3s, but it sits inside the SAME `Promise.all` as two much
slower calls вҖ” `/api/v1/paper/orders` (~6.2s) and `/api/v1/portfolio/f-auto`
(~7.4s) вҖ” so the capital section (and the "йңҖиҰҒ Owner зҷ»е…ҘжүҚиғҪй җиҰҪ / йҖҒеҮәзҙҷдёҠе–®"
copy attached to it) stayed locked for ~7-10s on *every* load, handoff or
not. It is not a cookie/session bug вҖ” it is a perceived-latency + wrong-copy
bug: "not fetched yet" was unconditionally labeled as "not authorized."

Confirmed via timed repro against prod (3 scenarios, same ~7-8s each):
`no-handoff-2330`, `handoff-2330-held`, `handoff-3707-new`.

### Fix (frontend only, `apps/web/lib/final-v031-live.ts`)
1. `fetchCapitalFast()` вҖ” a new, standalone `GET /api/v1/paper/portfolio`
   fetch fired alongside (not instead of) the full refresh, decoupling
   capital/ticket readiness from the slow symbol-specific waterfall
   (company вҶ’ quote вҶ’ OHLCV вҶ’ optional KGI bidask/ticks) and the two slow
   calls above. Guarded so it never overwrites data the full refresh already
   landed.
2. `baseCapitalUnauthorized` вҖ” new explicit flag, only set on a genuine
   401/403 on the portfolio fetch itself. The ordinary "not fetched yet"
   state now renders honest "ијүе…Ҙдёӯ" / "жң¬йҮ‘иіҮж–ҷијүе…ҘдёӯпјҢи«ӢзЁҚеҖҷ" copy instead of
   "еҫ…жҺҲж¬Ҡ" / "йңҖиҰҒ Owner зҷ»е…ҘжүҚиғҪй җиҰҪ / йҖҒеҮәзҙҷдёҠе–®" (that copy is preserved for
   the real-unauthorized case).

### Verified (local `next start` build wired to prod API + real Owner session, not yet deployed)

## P0-4 вҖ” paper еёідёҚеӢҫзЁҪ

### Root cause
`GET /api/v1/paper/portfolio` keeps a row for every symbol ever traded,
including ones that net back to 0 shares (a closed round-trip вҖ” backend
marks these `note:"net_flat_or_short"`, `avgCostPerShare:null`). The
frontend counted every row (open or closed) as a "held position," producing
the exact contradiction the critique caught: "жҢҒжңү 1 жӘ”жЁЎж“¬йғЁдҪҚ" next to
"жҢҒеҖүеёӮеҖј 0" вҖ” the 0 was mathematically correct for a flat position, the "1
held" label was not.

Separately, `еҸҜз”ЁиіҮйҮ‘` always rendered the same number as `жЁЎж“¬жң¬йҮ‘`
(`summary.baseCapitalTWD`, a fixed `PAPER_BROKER_INITIAL_CASH` constant,
never adjusted for trades) вҖ” the backend already computes
`summary.investedCostTWD` (cost basis tied up in open positions) but the
frontend never used it.

### Fix (frontend only, `apps/web/lib/final-v031-live.ts`)
1. `openPositions = portfolio.filter(pos => netQtyShares > 0)` вҖ” every
   reconciled summary stat (жҢҒжңү N жӘ”пјҸжҢҒеҖүеёӮеҖјпјҸзёҪжҗҚзӣҠпјҸжЁЎж“¬еә«еӯҳ badge) now
   uses this subset, not the raw array. A fully-closed round trip now
   correctly shows "0 held / вҖ” mktval / вҖ” pnl" instead of "1 held / 0
   mktval." The raw ledger table (жЁЎж“¬еә«еӯҳ tab) still lists closed rows as
   history вҖ” that's a legitimate record, not a summary claim вҖ” but each
   closed row is now tagged "е·Іе№іеҖү" instead of presented as if still held.
2. еҸҜз”ЁиіҮйҮ‘ = `baseCapitalTWD вҲ’ investedCostTWD` (backend-computed, additive
   field threaded through `clientPaperPayload()` + `fetchCapitalFast()` +
   the SSR fastShell default).
3. Cost-basis-substituted prices вҖ” any open position that isn't the symbol
   currently on screen has no live quote fetched for it and falls back to
   `avgCostPerShare` as a price stand-in. That fallback already existed;
   it's now tagged "д»ҘжҲҗжң¬дј°" (mirrors #1149's F-AUTO жңӘиЁҲеғ№/д»ҘжҲҗжң¬дј°
   pattern) instead of silently presented as live, in both the summary
   aggregate and the per-row ledger table.

### Known remaining gap (escalated, not silently hidden)
`GET /api/v1/paper/portfolio` does not track realized P&L from closed
round-trips вҖ” `baseCapitalTWD` is a static constant, so a closed trade's
proceeds are never reflected back into cash. Fully solving that needs a
backend ledger change (apps/api, out of this fix's frontend-only scope,
different from the durable F-AUTO SIM ledger which does track this).
Flagging for Elva/Jason rather than pretending the frontend fix makes the
number complete.
