# IUF 每日決策流產品設計 v1 — 登入到建立今日交易判斷（Fable 5 定版 2026-07-03）

**題目**（Codex 提案、楊董轉派）：使用者從登入後到建立今日交易判斷的完整產品流 — 哪些頁面該留、串起來、弱化或合併。
**地位**：交易台 epic 之外的產品主軸設計；#1125 today cockpit DRAFT 的正式處置；Codex 實作的上游規格。
**分工**（照 Codex 建議採納）：本檔=方向/規格/文案原則；Codex=落地實作；Jim=小件。

---
## §1 診斷：零件都在，流是斷的

現況使用者旅程（實際）：登入 → 戰情台看一眼 → **自己記得**要去看晨報 → **自己記得**去 AI 推薦 → 看到有興趣的 → **手動**去公司頁查 → 查完 → **手動**去交易室 → **重新輸入**代號下單。每一步的 context 全靠使用者腦內攜帶，頁面之間零狀態傳遞。
- 死路頁：訊號看完沒有下一步；主題看完不知道買誰；復盤看完不知道改什麼。
- #1125 cockpit DRAFT 卡 11 天的根因：它想用「再加一頁」解決，但問題不是缺頁，是**既有頁之間缺動線**。

**IA-2 決策流主軸（三步，每步有明確出口）**
`①今天什麼狀況（戰情台）→ ②值得看什麼（推薦/情報/公司頁）→ ③要不要行動（交易室/觀察/計畫）`
設計鐵律：**每一張卡片、每一個列表項，都要能回答「所以我下一步按哪裡」** — 沒有 CTA 的資訊卡=設計不完整。

## §3 戰情台 → 決策發射台（C-1，最大件）

三區塊直排（行動版同序）：
1. **今日狀態列**：大盤（即時/收盤誠實標）｜我的倉位一行（總資產、今日損益、曝險 %，數字勾稽鐵律照舊）｜今日是否交易日/開盤倒數。
2. **今日必看**（≤5 張卡）：晨報摘要卡（點入 /briefs）＋重大訊息 top（點入公司頁）＋風控警示（有才顯）。
3. **今日機會 → 行動**：AI 推薦 top3（每張卡三鍵：看公司／加觀察／帶入下單 ticket）＋自選異動。
- 資料需求：**一支聚合端點 `GET /api/v1/cockpit/today`**（欄位：market_state、portfolio_summary、brief_teaser、top_alerts、top_picks、watchlist_movers）— 全部是既有資料的重組，零新資料源；一次呼叫解決首頁 N 連發。
- ⚠️ 首頁改版=品味題：**Codex 先出 wireframe/截圖，楊董拍板才動工**（比照公司頁 Round 2 流程）。

## §4 連動規則（C-2，Codex 第 2 點）

| 從 | 到 | 具體元素 |
|---|---|---|
| 公司頁 | 所屬主題 | 主題 chips（點入主題頁） |
| 主題頁 | 受惠公司 | 成員列每行：現價＋今日漲跌＋「加觀察」鍵 |
| AI 推薦卡/訊號卡 | 行動 | 固定三鍵：**看公司／加觀察／帶入模擬單**（ticket 預填代號— 依 F2 統一下單流，等 PR-3 後接） |
| 訊號卡 | 誠實度 | 必標訊號時間戳＋「N 分鐘前」；過期訊號灰顯不誘導 |
| 復盤頁 | 改進 | 每週檢討尾部固定「本週最大失誤 → 對應哪個設定/習慣」一行（有資料才顯，不硬編） |

## §5 誠實狀態語氣統一（C-3，Codex 第 3 點 — 全站四態詞彙表）

| 態 | 視覺 | 文案模板 | 禁止 |
|---|---|---|---|
| 即時 | 綠點 | 「即時」或「盤中快照（約 5-20 秒）」 | 收盤資料標即時 |
| 收盤快照 | 琥珀 | 「MM/DD 收盤」（用資料自身日期） | 「今日收盤」配昨日資料 |
| 延遲/部分 | 琥珀 | 「資料延遲：<原因>」／「N/M 檔已計價，缺價部位以成本列示」 | 缺價當 0 混總額 |
| 無資料 | 灰 | 「尚無資料：<為什麼>＋<何時會有>」 | 永駐「載入中…」、空白區塊 |
- 落地：抽 `packages/ui` 共用 `<DataStateBadge>` 元件＋文案常數檔；**新頁強制用、舊頁逐頁換**（掃描清單由 sonnet 產）。

## §6 信任建立（C-4，Codex 第 4 點 — 券商連線頁）

固定「你的憑證在哪裡」說明卡（配對 UI #1151 已有一句，升級成卡）：①憑證只存在你自己的電腦，永不上傳 ②gateway 是你電腦上的小程式，我們只收「它還活著」的心跳 ③四態白話（未配對=還沒裝／等待=裝了還沒認親／已連線=一切正常／失聯=你電腦上的程式停了）④失敗自救三步（重開 gateway → 重發配對碼 → 找我們）。真金上線後（Phase 4）此卡加一節「每一筆真單都需要你本人密碼確認」。
