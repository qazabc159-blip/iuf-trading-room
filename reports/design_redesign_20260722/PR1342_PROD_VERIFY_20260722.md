## 3. 三大法人「同步中」根因鑑識（非本次改版回歸）

**現象**：初次 fresh-login 訪問 + 連續 3 次同 session `page.reload()`（共 4 次觀測，跨約 3-4
分鐘），三大法人 Panel **持續**顯示「同步中 / --」，非本次改版新增的「一次性冷啟動暫態」
（不同於昨晚 #1338 驗證時見過的、reload 一次就恢復的暫態空白）。

**直接查後端**：
```
GET /api/v1/market/institutional-summary/finmind
→ {"asOf":null,"totalNet":null,"institutions":[],"topNetBuy":[],"topNetSell":[],
   "source":"finmind","staleAfterSec":60,"state":"unavailable","reason":"finmind_returned_empty"}
```
後端誠實回報 `state=unavailable, reason=finmind_returned_empty`——`institutions[]` 本身是空陣列，
不是分類邏輯匹配失敗（#1338 的分類 regex 本次改版未動，commit message 已註明「no data-layer
change」）。

**交叉驗證排除「全站故障」**：同時查 `GET /api/v1/market/leaders/finmind`（另一支 FinMind 來源
端點）**同樣回空**且同一 `reason=finmind_returned_empty`；但同頁的熱力圖（`TWSE 公開資料`來源，
非 FinMind）**同時段有真實 12 磚數值**。三者合觀，確認是 **FinMind 這條資料管線本身此刻（跨
午夜 00:4x TST）處於空窗**，與 TWSE 來源無關、與 #1342 前端改版無關、與 #1338 的分類修復無關。
前端 `if (!institutional) return null` 分支未觸發（`institutional` 物件本身非 null，是
`institutionalRaw.institutions` 內容為空使各 bucket 的 `line()` 回傳 null），改走每列各自的
「同步中」/`--` 誠實降級呈現——**這正是設計要的行為（不造假、不崩潰），不是 bug**。

**結論**：無法在本次驗證窗口親眼確認「三大法人三列真值」這一項（因為此刻後端真的沒有資料可顯示），
但已排除是 #1342 或 #1338 的回歸；建議白天交易時段或 FinMind 資料補齊後再複驗一次這一項即可，
不影響本次改版其餘 6 項驗收結果。
