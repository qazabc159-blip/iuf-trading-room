## ISSUE_001: Company Orderbook 404

**Root cause**: `GET /api/v1/companies/:id/orderbook` did not exist.

**Fix**: Added route (Owner-gated). Source chain:
1. Primary — KGI gateway `/bidask` (enforces 40-cap whitelist, calls `subscribeSymbolBidAsk` + `getLatestBidAsk`)
2. Fallback — `_orderbookLastKnownCache` (in-memory Map, keyed by symbol)
3. Off-hours — `composeTaiwanMarketState().state === "POST-CLOSE"` → returns `{state:"off_hours", last_close:<X from companies_ohlcv>, note:"盤後 · 等明日開盤"}`

**Response shape**:
```json
{
  "data": {
    "symbol": "2330",
    "state": "LIVE|STALE|LAST_KNOWN|off_hours|BLOCKED|NO_DATA",
    "bids": [{"price": 1050, "volume": 100}],
    "asks": [{"price": 1051, "volume": 200}],
    "source": "kgi-gateway|last_known|off_hours",
    "last_close": null,
    "note": null,
    "updatedAt": "2026-05-14T..."
  }
}
```

**Added exports** (tests only): `_resetOrderbookLastKnownCache()`

## ISSUE_004: Brief list heading=''

**Root cause**: `/api/v1/briefs` returned raw `DailyBrief[]` from `repo.listBriefs()`.
`DailyBrief` schema has no top-level `heading` field — only `sections[0].heading`.
Frontend `.heading` read always got `undefined` (rendered as empty string).

**Fix**: Expanded the one-liner handler to map over results and project
`heading: sections[0]?.heading ?? "Brief {date}"` into each item.
